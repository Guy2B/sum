# Sigma Life OS V4.9.1 — Meta Connector

## Included

- Secure Meta OAuth through Firebase Cloud Functions.
- Facebook Pages discovery.
- Instagram Business/Creator account discovery when linked to a Page.
- Import of recent Page/Instagram publications and comments.
- Private token storage in server-only Firestore paths.
- Meta webhook verification endpoint and event intake.
- No automatic publishing in this release.

## Required Meta configuration

1. Create a Meta developer app.
2. Add Facebook Login for Business / the products required by your Meta setup.
3. Configure the exact OAuth callback URL shown by Firebase after deploying `metaOAuthCallback`.
4. Request only the permissions your use case requires. Advanced access and App Review may be required by Meta.
5. Add your Facebook/Instagram test users while the app is in development mode.

## Firebase secrets and parameters

From the project root:

```bash
firebase functions:secrets:set META_APP_SECRET
firebase functions:secrets:set META_WEBHOOK_VERIFY_TOKEN
firebase functions:config:set meta.app_id="YOUR_META_APP_ID"
```

This release uses Firebase parameterized configuration. During `firebase deploy --only functions`, provide:

- `META_APP_ID`
- `META_REDIRECT_URI` (the deployed `metaOAuthCallback` URL)
- `META_PUBLIC_APP_URL` (`https://guy2b.github.io/sum/app.html`)
- `META_GRAPH_VERSION` (keep this aligned with the Graph API version selected in Meta)

Then set in `social-config.js`:

```js
meta: {
  enabled: true,
  configured: true,
  appId: 'YOUR_PUBLIC_META_APP_ID'
}
```

The Meta App Secret must never be placed in GitHub Pages or `social-config.js`.

## Deploy

```bash
cd functions
npm install
cd ..
firebase deploy --only functions,firestore:rules
```

Upload the complete V4.9.1 project to GitHub, commit, and push.
