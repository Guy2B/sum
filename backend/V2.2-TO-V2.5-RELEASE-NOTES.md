# Sigma Life OS V2.5 Beta — Connected Platforms Edition

Cette archive est cumulative et couvre les jalons V2.2 à V2.5.

## V2.2 — Google Workspace réel
- Gmail : lecture des messages récents.
- Gmail : envoi de messages via `gmail.send`.
- Google Calendar : import des événements.
- Google Calendar : création d’événements et invitations.
- Google Drive : sauvegarde/restauration dans `appDataFolder`.
- YouTube : lecture des abonnements.
- Un seul Client ID OAuth Web, sans secret dans le frontend.

## V2.3 — Gemini et automatisations assistées
- Proxy Google Apps Script Gemini conservé.
- Réécriture et enrichissement des réponses du coach.
- Aucun secret Gemini dans GitHub Pages.
- Les traitements automatiques quand le navigateur est fermé restent hors périmètre sans backend planifié.

## V2.4 — Réseaux et messageries externes
- Registre de capacités pour Meta, LinkedIn, X, TikTok et Microsoft.
- Fichier `platform-config.js` pour les identifiants publics.
- Les connexions réelles nécessitent les applications développeur et validations de chaque plateforme.
- X et certains flux de publication nécessitent un backend et peuvent dépendre d’offres payantes.

## V2.5 — Santé et montres
- Pont santé web/native existant conservé.
- Architecture prête pour Health Connect, Wear OS, HealthKit et Apple Watch.
- Les données de montre ne peuvent pas être lues directement par GitHub Pages : les apps Android/iOS natives restent obligatoires.

## Configuration requise
1. Compléter `firebase-config.js`.
2. Compléter `google-cloud-config.js` avec le nouveau Client ID OAuth.
3. Activer Gmail API, Calendar API, Drive API et YouTube Data API v3.
4. Ajouter `https://guy2b.github.io` aux origines JavaScript autorisées.
5. Publier `firestore.rules`.
6. Pour Gemini, déployer `backend/google-apps-script-gemini` et renseigner l’URL `/exec`.

## Limites honnêtes
- L’OAuth Gmail/Calendar/Drive fonctionne tant que la page est ouverte; aucun rafraîchissement en arrière-plan navigateur fermé.
- Meta, LinkedIn, TikTok et X ne deviennent pas opérationnels sans comptes développeur, permissions et validation fournisseur.
- Apple Health/Watch et Health Connect/Wear OS exigent une application mobile native signée.
