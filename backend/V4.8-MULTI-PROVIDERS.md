# Sigma Life OS V4.8 — Multi Providers

V4.8 reprend tout Google Complete de V4.7 et ajoute une gestion par écosystème.

## Microsoft / Outlook (OAuth réel)

1. Dans Microsoft Entra admin center, créez une **App registration** nommée Sigma Life OS.
2. Comptes pris en charge : comptes professionnels et comptes Microsoft personnels si souhaité.
3. Dans **Authentication**, ajoutez la plateforme **Single-page application** avec :
   `https://guy2b.github.io/sum/app.html`
4. Dans **API permissions > Microsoft Graph > Delegated permissions**, ajoutez :
   - User.Read
   - Mail.Read
   - Mail.Send
   - Calendars.ReadWrite
   - Contacts.Read
   - Files.ReadWrite.AppFolder
5. Copiez l'**Application (client) ID** dans `platform-config.js` :

```javascript
microsoftClientId: 'VOTRE-CLIENT-ID'
```

Ne mettez aucun secret client dans GitHub. La SPA utilise MSAL et le flux Authorization Code + PKCE.

## Yahoo, GMX, iCloud et IMAP

Ces fournisseurs ne peuvent pas être joints directement depuis GitHub Pages par IMAP/SMTP. V4.8 affiche leur assistant et réutilise `backend/mail-connector`, mais vous devez déployer ce connecteur sur Firebase/Cloud Run, puis renseigner :

```javascript
mailConnectorBaseUrl: 'https://votre-connecteur.example'
```

Les mots de passe d'application doivent être transmis uniquement au connecteur HTTPS et chiffrés côté serveur. Ne les placez jamais dans `platform-config.js`, `app.js`, Firestore en clair ou GitHub.

## État honnête

- Google : APIs navigateur intégrées.
- Microsoft : OAuth/Graph intégrés après ajout du Client ID.
- Yahoo/GMX/iCloud/IMAP : interface et contrat de connecteur prêts; déploiement serveur requis.
