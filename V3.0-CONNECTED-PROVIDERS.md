# Sigma Life OS V3.0 — Connected Providers

## Phases 1 à 6

1. **Google** — compte Sigma unifié, consentement Gmail séparé, import réel Gmail, Calendar/Drive via le module Google existant.
2. **Microsoft** — carte fournisseur et configuration Entra ID. La synchronisation Outlook/Calendar/OneDrive nécessite le connecteur Microsoft Graph.
3. **Apple** — iCloud Mail via IMAP et mot de passe spécifique à l’application. Apple Health reste natif iOS.
4. **Yahoo** — IMAP via mot de passe d’application et connecteur sécurisé.
5. **Proton** — Proton Mail Bridge ou connecteur compatible requis.
6. **IMAP universel** — GMX, OVH, Infomaniak, IONOS, Zoho et domaines personnalisés via le connecteur serveur.

## Principe de sécurité

GitHub Pages ne stocke jamais les mots de passe principaux. Google fonctionne directement par OAuth dans le navigateur. Les fournisseurs IMAP doivent passer par un backend HTTPS configuré dans `platform-config.js` (`mailConnectorBaseUrl`).

## Configuration

- Conserver `firebase-config.js` et `google-cloud-config.js` avec les valeurs du projet.
- Configurer `microsoftClientId` pour préparer Microsoft.
- Configurer `mailConnectorBaseUrl` pour Yahoo, Apple/iCloud, Proton Bridge et IMAP.
- Ne jamais publier de secret client, mot de passe d’application ou compte de service.

## Migration V2.5

Au premier chargement, les comptes et messages de démonstration sont supprimés. Le bouton Google utilise l’adresse du compte Sigma connecté et demande le consentement Gmail au premier usage.
